# yaml

## How do you represent an array/list?

<!-- notecardId: 1761633867477 -->

```yaml
fruits:
-   orange
-   apple
-   banana
```

## How do you represent a dictionary/map?

<!-- notecardId: 1761634868502 -->

```yaml
banana:
    calories: 105
    fat: 0.4 g
    carbs: 48 g
```

## How do you represent key value pair?

<!-- notecardId: 1761634868505 -->

```yaml
fruit: apple
vegetable: carrot
meat: chicken
```

## How do you represent a dictionary within another dictionary?

<!-- notecardId: 1761634868507 -->

```yaml
car: 
    color: blue
    model:
        name: corvette
        year: 1995
    transmission: manual
```

## How do you represent a list of dictionaries?

<!-- notecardId: 1761634868510 -->

```yaml
-   color: blue
    model: 
        name: corvette
        year: 1995
    transmission: manual
-   color: grey
    model: 
        name: corvette
        year: 1995
    transmission: automatic
```

## For what data structures does order of items matter?

<!-- notecardId: 1761634868512 -->

array/list

## How do you comment in yaml?

<!-- notecardId: 1761634868514 -->

with \#
